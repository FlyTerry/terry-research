## 数据准确性原则
- 以原始财报 PDF 为优先来源，标注页码；
- 三态标记：`✅ 已核验` / `⚠️ 估算` / `🚫 未披露`；
- 跨报告同口径数据须一致，修订时同步更新源数据与报告。

## 历史财务数据提取（从年报 PDF 直提，可复用）
> 各公司 `历史财务数据.md` 即由以下脚本生成，用 `pdfplumber` 解析年报 PDF，跨相邻年报比较栏交叉验证。
- `tools/extract_cn_financials.py`：从 A 股年报**合并利润表/资产负债表**提取营收/营业成本/归母净利/EPS/归母权益（鲁棒版，`require` 锁定真实报表页，剔除"主要会计数据"摘要页）。
- `tools/extract_summary_financials.py`：从年报**「主要会计数据和财务指标」文本摘要表**提取官方口径营收/归母净利/EPS/加权ROE/归母净资产/总资产；每份年报含 3 个年度，跨报告串联 2021–2025 并交叉验证。
- `tools/compile_cn_financials.py <公司目录>`：编译某公司 2021–2025 序列（营收/成本/毛利率/净利率/EPS/权益/ROE）并输出交叉验证结果，`import` 上述提取器。
- 运行环境：`~/.workbuddy/binaries/python/envs/default`（已装 pdfplumber）。**注**：紫金合并报表为图片版，走 `extract_summary_financials.py`；其余 A 股走利润表直提 + 摘要表补 ROE/总资产。
- **复核报告**：`data/财务数据复核报告.md` — 2026-07-11 对 4 家 `历史财务数据.md` 二次独立复核（重新直提 + 跨报告交叉验证 + 关键项溯源），结论：全部与官方年报一致，无错误项。
- **储量/产量/权益复核报告**：`data/储量产量权益复核报告.md` — 2026-07-11 对 5 家报告 HTML 的储量/产量/权益主张数据，用 2025 年报 PDF 直提逐项核对。结论：5 份报告均存在偏差，共约 33 处需修订（数值错误/口径混淆/误标未披露/系统性颠倒），修订清单见该报告，待确认后执行 HTML 修订。
