# Terry Research 数据与设计风格规范

> 生效日期：2026-07-11 ｜ 适用范围：所有研究报告（煤炭 / 有色金属）
> 配套文件：`assets/terry-style.css`（统一组件样式）、`assets/template.html`（报告骨架）

本规范用于统一全站报告的**视觉设计**与**数据表达**，确保跨行业、跨公司报告风格一致、可读、可比对。

---

## 一、设计系统（视觉）

### 1.1 行业强调色（唯一随行业切换的变量）

| 行业 | body class | 强调色 `--brand` | 辅助色 `--brand-2` |
|------|-----------|------------------|---------------------|
| 煤炭 | `theme-coal` | 蓝 `#185FA5` | 青绿 `#0F6E56` |
| 有色金属 | `theme-nonferrous` | 金 `#C9952B` | 深蓝 `#1A3E72` |

所有其他组件（字体、间距、表格、导航、框、状态标记）在全站完全一致，不因行业改变。

### 1.2 统一组件（来自 `terry-style.css`）

- 顶部导航 `.site-header`：固定吸顶，含站点 logo（返回首页）+ 当前报告面包屑
- 报告标题 `.report-title` + 版本徽标 `.version-badge`
- 章节 `.chapter` + `h2`（左侧色条）/ `h3` / `h4`
- 表格 `table` + `.table-wrap`（宽表横向滚动）+ `.num`（数值右对齐）+ `.total`（汇总行）
- 信息框 `.box` / `.box--info` / `.box--note` / `.box--warning` / `.box--danger`
- 数据来源行 `.data-source`
- 状态标记 `.m-ok` / `.m-warn` / `.m-no`，标签 `.tag` / `.tag--ok/--warn/--no`
- KPI 卡 `.kpi-grid` + `.kpi`
- 详情卡 `.detail-card`、目录 `.toc`、图表 `.chart-box`、页脚 `.site-footer`

> ⚠️ 各报告**不再内联 `<style>`**，统一 `<link rel="stylesheet" href="assets/terry-style.css">`。
> 新增样式请优先扩展共享 CSS，保持全站一致。

## 二、数据表达规范

### 2.1 数值与单位

| 项目 | 规范 | 示例 |
|------|------|------|
| 金额（A股/港股公司） | 优先用「亿元」，并标注币种；原始财报千元数据换算后标注 | `180.56 亿元`（人民币） |
| 千分位 | 四位及以上整数加千分位逗号 | `18,055,551` / `47,011` |
| 百分比 | 涨跌用 `+` / `−` 号，保留 1 位小数 | `+56.3%` / `−7.0%` |
| 产量/产能 | 带单位，量词用中文或国际单位 | `4,000 吨/日`、`47,011 千克` |
| 储量/资源量 | 金属量用「吨/千克/盎司」，矿石量用「吨」 | `312.0 吨` |
| 价格 | 标注币种与口径 | `USD 2,350 / 盎司` |

- 数值列在表格中统一加 `.num` 实现右对齐 + 等宽数字。
- 基本面指标涨跌着色（研究报告惯例）：`.positive`=绿（利好/增长）、`.negative`=红（利空/下降）、`.flat`=灰（持平）。
  > 注：A 股行情「红涨绿跌」仅用于实时股价 tick；本研究报告聚焦公司基本面，统一采用「绿=利好、红=风险」。

### 2.2 数据来源与核验标记（全站统一三态）

| 标记 | 含义 | 用法 |
|------|------|------|
| `✅ 已核验` / `.m-ok` | 来自官方/权威渠道，已核对 | 年报、官网、交易所披露 |
| `⚠️ 估算` / `.m-warn` | 基于公开信息的推算值 | 年化产能、间接测算 |
| `🚫 未披露` / `.m-no` | 公司未公开披露 | 单矿储量、品位、成本等 |

- 来源标注统一放在表格末列，或独立 `.data-source` 行，并尽量附原始链接。
- 未披露项写「公司未公开披露 / 公司未披露」，不使用模糊表述。

### 2.3 标题与命名

- 报告 H1：`<报告名>（起始年–结束年）`，**不加 emoji 前缀**；版本号用 `.version-badge`（如「详细版 v1.0」），不写进标题正文。
- 年区间使用 en dash `–`（非连字符、非破折号）：`（2021–2025年）`。
- 章节标题用「第 N 章 标题」，与目录 `.toc` 锚点一致。
- 公司报告文件名英文、报告内容中文（既有约定保留）。

### 2.4 数据准确性

- 关键财务数据以**原始财报 PDF** 为优先来源，并在 `.data-source` 标注页码。
- 跨报告同口径数据应一致；若修订，需在对应报告与 `data/` 源数据同步更新。

## 三、文件与目录结构

```
terry-research/
├── index.html              # 首页（统一系统）
├── assets/
│   ├── terry-style.css     # 共享设计系统（本规范样式实现）
│   └── template.html       # 报告骨架模板
├── data/                   # 源数据（markdown，按公司/行业归档）
├── docs/                   # 规范与说明文档
├── tools/                  # 数据提取/更新脚本（Python）
└── *-report.html           # 各研究报告
```

---

## 四、新增 / 修订报告清单

1. `<head>` 引入 `assets/terry-style.css`，`body` 标注 `theme-coal` 或 `theme-nonferrous`。
2. 套用 `.site-header` 导航 + `.report-title` + `.toc` + `.site-footer` 结构。
3. 表格用 `.table-wrap` + `.num` + `.total`；数据来源用 `.data-source` + 三态标记。
4. 数据写入 `data/` 对应文件，并标注来源与页码。
5. 在 `index.html` 登记报告入口。
